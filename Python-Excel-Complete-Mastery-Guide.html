Chapter 1: The Python & Excel Universe üåç

Understanding the Landscape

Right, before we write any code, let's understand what we're dealing with. Python and Excel are like two different languages that can work brilliantly together once you learn to translate between them.

What Is Python?

Python is a programming language - a way to give instructions to your computer. Unlike Excel where you click buttons, in Python you write text commands.

üéØ Why Python Is Perfect for Excel Users

  • Similar concepts: Excel has cells and formulas, Python has variables and functions
  • Readable syntax: Python code reads almost like English
  • Pandas library: Works just like Excel tables
  • Massive community: Any problem you have, someone's solved it
  • Free forever: No licences, no subscriptions

Python vs VBA vs Power Query

Let me clear up the confusion:

Tool Best For Limitations
VBA Excel-specific automation, interacting with Excel UI Only works in Excel, outdated language, slow
Power Query Data transformation within Excel, GUI-based Limited to Excel, can't handle complex logic
Python Everything! Complex automation, data science, web scraping, ML Steeper learning curve (but worth it)

The Python Ecosystem for Excel

Python has different "libraries" (think of them as add-ons) for working with Excel:

üì¶ Key Python Libraries

  • pandas: Data manipulation (like Excel on steroids)
  • openpyxl: Read/write .xlsx files without Excel installed
  • xlrd/xlwt: Work with older .xls files
  • xlwings: Control Excel application from Python (like VBA)
  • pywin32: Windows COM automation (advanced Excel control)

Setting Up Your Environment

We need to install Python on your computer. Follow these steps:

Step 1: Install Python

  1. Go to python.org
  2. Download Python 3.11+ (or latest version)
  3. CRITICAL: During installation, tick "Add Python to PATH"
  4. Click "Install Now"
  5. Wait for installation to complete

üö® Common Installation Mistake

If you forget to tick "Add Python to PATH", Python won't work from command line. If this happens, uninstall and reinstall, making sure to tick that box!

Step 2: Verify Installation

  1. Open Command Prompt (Windows) or Terminal (Mac/Linux)
  2. Type: python --version
  3. You should see something like: Python 3.11.5

Step 3: Install VS Code (Recommended Editor)

You need somewhere to write Python code. Visual Studio Code is free and brilliant:

  1. Go to code.visualstudio.com
  2. Download and install
  3. Open VS Code
  4. Click Extensions (left sidebar, square icon)
  5. Search for "Python" and install Microsoft's Python extension

Step 4: Install Essential Libraries

Open Command Prompt / Terminal and type these commands:

# Install pandas (data manipulation) pip install pandas # Install openpyxl (Excel file handling) pip install openpyxl # Install xlwings (Excel automation) pip install xlwings # Install matplotlib (visualisation) pip install matplotlib

Wait for each to finish installing before moving to the next.

Your First Python Program

Let's write actual code! Open VS Code and create a new file called hello.py:

# This is your first Python program! print("Hello, Excel World!")

To run it:

  1. Save the file
  2. Open terminal in VS Code (Terminal ‚Üí New Terminal)
  3. Type: python hello.py
  4. You should see: Hello, Excel World!

üéâ Congratulations!

You've just written and run your first Python program. Simple, wasn't it? This print() function is like VBA's MsgBox or Debug.Print - it displays output.

Understanding Python Syntax Basics

Python is fussy about indentation (spaces). This will trip you up initially:

# CORRECT - consistent indentation if 5 > 3: print("Five is greater") print("This line is also indented") # WRONG - inconsistent indentation (will error!) if 5 > 3: print("Five is greater") print("Extra spaces cause errors")

üí° Indentation Best Practice

Use 4 spaces for each indentation level. VS Code does this automatically when you press Tab. Never mix tabs and spaces!

Python vs Excel: Translation Guide

Here's how Excel concepts map to Python:

Excel Concept Python Equivalent Example
Cell (A1) Variable sales = 1000
Formula (=SUM(A1:A10)) Function sum(values)
Range (A1:B10) List/Array [1, 2, 3, 4]
Table DataFrame (pandas) df = pd.DataFrame()

Key Python Concepts (Quick Overview)

Variables - Like Excel Cells

# Storing values (like typing in Excel cells) name = "Alice" age = 30 salary = 50000.50 is_manager = True

Data Types - Like Excel Cell Formats

# Different types of data text_value = "Hello" # String (text) number_value = 42 # Integer (whole number) decimal_value = 3.14 # Float (decimal) true_false = True # Boolean (TRUE/FALSE in Excel)

Lists - Like Excel Ranges

# Lists hold multiple values (like A1:A5 in Excel) sales = [1000, 1500, 2000, 1800, 2200] products = ["Widget A", "Widget B", "Widget C"] # Access items (like referring to cell A1, A2, etc.) first_sale = sales[0] # Gets 1000 (Python counts from 0!) second_sale = sales[1] # Gets 1500

üö® Python Counts from Zero!

This trips up everyone initially. In Excel, the first row is Row 1. In Python, the first item is index 0:

  • Excel: Row 1, 2, 3, 4...
  • Python: Index 0, 1, 2, 3...

Comments - Documenting Your Code

# This is a single-line comment (like Excel comment) """ This is a multi-line comment. Useful for longer explanations. Like VBA's apostrophe comments. """ salary = 50000 # You can also add comments at end of lines

Chapter 2: Python Fundamentals üêç

Right, now we're going to properly learn Python. Think of this chapter as your Python driving lessons - we'll start slow and build up. By the end, you'll be comfortable with Python basics.

Variables and Assignment

In Excel, you put values in cells. In Python, you put values in variables:

# Simple assignment (like typing value in cell) price = 29.99 quantity = 5 total = price * quantity print(total) # Output: 149.95 # Variables can be reassigned (like changing cell value) price = 34.99 print(price) # Output: 34.99

üìä Excel Analogy

Think of variables as named cells:

  • price = 29.99 is like typing 29.99 in cell A1
  • quantity = 5 is like typing 5 in cell A2
  • total = price * quantity is like formula =A1*A2

Naming Conventions

Python has rules about naming variables:

# GOOD variable names customer_name = "John Smith" total_sales = 15000 is_active = True # BAD variable names (will cause errors) # 2nd_customer = "Invalid" # Can't start with number # customer-name = "Invalid" # Can't use hyphens # for = "Invalid" # Can't use Python keywords # Technically valid but not recommended x = 100 # What is x? Be descriptive! data = "Too vague" # What kind of data?

üí° Naming Best Practices

  • Use snake_case for variables: customer_name not CustomerName
  • Be descriptive: total_revenue not tr
  • Use lowercase for variables
  • Use UPPERCASE for constants: VAT_RATE = 0.20

Basic Data Types

1. Strings (Text)

# Strings can use single or double quotes name = "Alice Jones" city = 'London' # String operations full_name = "Alice" + " " + "Jones" # Concatenation greeting = "Hello" * 3 # "HelloHelloHello" # Useful string methods message = " Sales Report " clean_message = message.strip() # Removes spaces (like TRIM in Excel) upper_message = message.upper() # SALES REPORT (like UPPER in Excel) lower_message = message.lower() # sales report (like LOWER in Excel) # String formatting (modern way) name = "Alice" age = 30 text = f"{name} is {age} years old" # f-strings print(text) # Output: Alice is 30 years old

2. Numbers

# Integers (whole numbers) age = 30 quantity = 100 # Floats (decimals) price = 29.99 vat_rate = 0.20 # Mathematical operations total = 10 + 5 # Addition: 15 diff = 10 - 5 # Subtraction: 5 product = 10 * 5 # Multiplication: 50 quotient = 10 / 5 # Division: 2.0 power = 10 ** 2 # Power: 100 remainder = 10 % 3 # Modulo (remainder): 1 # Useful for currency price = 29.999 rounded_price = round(price, 2) # 30.00

3. Booleans (True/False)

# Boolean values (like TRUE/FALSE in Excel) is_active = True is_manager = False # Comparison operations result = 10 > 5 # True result = 10 < 5 # False result = 10 == 10 # True (equal to) result = 10 != 5 # True (not equal to) result = 10 >= 10 # True (greater than or equal) # Logical operations (like AND, OR in Excel) result = True and False # False result = True or False # True result = not True # False

Lists - Python's Version of Excel Ranges

# Creating lists sales = [1000, 1500, 2000, 1800, 2200] products = ["Widget A", "Widget B", "Widget C"] mixed = [42, "Hello", True, 3.14] # Can mix types # Accessing items (remember: counting from 0!) first_sale = sales[0] # 1000 second_sale = sales[1] # 1500 last_sale = sales[-1] # 2200 (negative counts from end) # Slicing (like selecting ranges in Excel) first_three = sales[0:3] # [1000, 1500, 2000] last_two = sales[-2:] # [1800, 2200] # Adding to lists sales.append(2500) # Add to end sales.insert(0, 900) # Insert at position 0 # Removing from lists sales.remove(1500) # Remove specific value popped = sales.pop() # Remove and return last item # Useful list operations length = len(sales) # How many items? (like COUNTA in Excel) total = sum(sales) # Add them up (like SUM in Excel) average = sum(sales) / len(sales) # Average (like AVERAGE in Excel) maximum = max(sales) # Highest value (like MAX in Excel) minimum = min(sales) # Lowest value (like MIN in Excel)

üìä Lists vs Excel Ranges

Excel Python
A1:A5 (range) [1000, 1500, 2000, 1800, 2200]
A1 (first cell) sales[0]
COUNTA(A1:A5) len(sales)
SUM(A1:A5) sum(sales)

Dictionaries - Like Named Ranges

Dictionaries store key-value pairs. Think of them like VLOOKUPs:

# Creating a dictionary customer = { "name": "John Smith", "age": 35, "city": "London", "is_active": True } # Accessing values name = customer["name"] # "John Smith" age = customer["age"] # 35 # Adding/updating values customer["email"] = "john@example.com" # Add new key customer["age"] = 36 # Update existing # Checking if key exists if "email" in customer: print(customer["email"])

Control Flow: If Statements

Like Excel's IF function, but more powerful:

# Simple if statement (like Excel IF) age = 20 if age >= 18: print("Adult") else: print("Minor") # Multiple conditions (like Excel nested IFs) score = 75 if score >= 90: grade = "A" elif score >= 80: grade = "B" elif score >= 70: grade = "C" else: grade = "F" print(f"Grade: {grade}") # Complex conditions temperature = 25 is_raining = False if temperature > 20 and not is_raining: print("Perfect weather for a walk!")

Loops - Repeating Actions

For Loops - Like Copying Formula Down

# Loop through a list sales = [1000, 1500, 2000, 1800] for sale in sales: print(f"Sale amount: ¬£{sale}") # Loop with index (like row numbers) for i, sale in enumerate(sales): print(f"Sale {i+1}: ¬£{sale}") # Loop specific number of times for i in range(5): print(f"Iteration {i}") # 0, 1, 2, 3, 4 # Loop from 1 to 10 for i in range(1, 11): print(i)

While Loops - Loop Until Condition

# Keep going until condition is False count = 0 while count < 5: print(count) count += 1 # Same as count = count + 1 # Be careful of infinite loops! # while True: # This would run forever! # print("Never ends...")

Appendix C: Troubleshooting Guide üîß

Common Python Errors

SyntaxError: invalid syntax

Cause: Typo or incorrect Python syntax

Solutions:

  • Check for missing colons after if/for/def statements
  • Ensure matching brackets/quotes
  • Verify indentation is consistent

IndentationError

Cause: Inconsistent spaces/tabs

Solutions:

  • Use only spaces (4 per level)
  • Never mix tabs and spaces
  • Configure editor to convert tabs to spaces

KeyError

Cause: Dictionary key doesn't exist

Solution:

# Use .get() method instead value = dictionary.get("key", "default")

IndexError: list index out of range

Cause: Accessing list item that doesn't exist

Solution:

# Check length first if len(my_list) > 5: value = my_list[5]

Common Excel Integration Issues

File Not Found

Solutions:

  • Use absolute paths: C:\Users\Name\file.xlsx
  • Or use raw strings: r"C:\path\to\file.xlsx"
  • Check file exists with os.path.exists()

Permission Denied

Cause: File is open in Excel

Solution: Close the Excel file before running Python script

Pandas Common Issues

DataFrame KeyError

Cause: Column name doesn't exist or has leading/trailing spaces

Solutions:

# Check actual column names print(df.columns.tolist()) # Strip spaces from column names df.columns = df.columns.str.strip()
